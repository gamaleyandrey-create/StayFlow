# Contributing to StayFlow

## Branches

- `main` — stable version.
- `dev` — active development.
- `feature/*` — new features.
- `fix/*` — bug fixes.

## Commit Style

Examples:

```text
feat: add booking model
fix: correct task status flow
docs: update API documentation
```

## Pull Requests

Every PR should include:

- What changed
- Why it changed
- Screenshots if UI changed
- Test notes
