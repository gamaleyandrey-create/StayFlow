# StayFlow MVP Plan

## Sprint 1 — Foundation

Цель: подготовить рабочий backend и базу.

Задачи:

- NestJS setup
- PostgreSQL
- Prisma
- Auth
- Users
- Properties CRUD

Результат:

- можно создать пользователя;
- можно создать объект;
- можно получить список объектов.

---

## Sprint 2 — Bookings & Tasks

Цель: реализовать главную автоматизацию.

Задачи:

- Bookings CRUD
- Guest creation
- Guest link generation
- Auto-create cleaning task after checkout
- Tasks CRUD
- Task status update

Результат:

- Owner создает booking;
- система автоматически создает cleaning task.

---

## Sprint 3 — Owner App MVP

Цель: базовый интерфейс для управляющего.

Экраны:

- Login
- Dashboard
- Properties
- Property Details
- Bookings
- Add Booking
- Tasks
- Task Details

Результат:

- Owner может управлять базовыми операциями.

---

## Sprint 4 — Worker App MVP

Цель: дать исполнителю рабочий интерфейс.

Экраны:

- Login
- Today Tasks
- Task Details
- Accept Task
- Start Task
- Complete Task
- Upload Photos
- Earnings

Результат:

- Worker получает задачу, выполняет и отправляет отчет.

---

## Sprint 5 — Guest App MVP

Цель: гостевая ссылка.

Экраны:

- Welcome
- Check-in Info
- Apartment Guide
- Chat
- Report Problem
- Services

Результат:

- Guest получает инструкции и может сообщить о проблеме.

---

## Sprint 6 — Launch Pilot

Цель: тест в Испании.

Задачи:

- подключить 3–5 управляющих компаний;
- добавить 30–100 объектов;
- измерить использование;
- собрать обратную связь;
- исправить UX.

Результат:

- продукт готов к первым платящим клиентам.
